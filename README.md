# HR Analytics

Цей проєкт представляє собою автоматизовану аналітичну систему для дослідження даних щодо працівників (HR analytics). Платформа побудована на основі мікросервісної архітектури з використанням Docker та оркестрацією через Docker Compose.

---

## Структура проєкту

Проєкт розділено на незалежні модулі, кожен з яких виконується у власному ізольованому контейнері.

    open-data-ai-analytics/
    │
    ├── data/                      # Спільний том: Вхідні дані
    │   └── Employee.csv           # Початковий датасет
    │
    ├── data_load/                 # Сервіс: Завантаження даних у БД
    ├── data_quality_analysis/     # Сервіс: Перевірка якості даних
    ├── data_research/             # Сервіс: Статистичне дослідження
    ├── visualization/             # Сервіс: Генерація графіків
    │
    ├── web/                       # Сервіс: Веб-інтерфейс (FastAPI)
    │   ├── templates/             # HTML-шаблони
    │   └── app.py                 # Логіка веб-сервера
    │
    ├── reports/                   # Спільний том: JSON-звіти
    ├── plots/                     # Спільний том: Згенеровані графіки
    │
    ├── compose.yaml               # Файл оркестрації сервісів
    └── README.md                  # Документація проєкту

---

## Опис сервісів

Система складається з 5 основних контейнерів, які об'єднані в єдину мережу `hr_network`:

1. **`data_load`:** Зчитує `Employee.csv` зі спільного тому `/data` та завантажує записи в таблицю PostgreSQL.
2. **`data_quality_analysis`:** Підключається до бази даних, аналізує пропуски та дублікати. Зберігає результати у `quality_report.json` (спільний том `/reports`).
3. **`data_research`:** Розраховує базові статистичні метрики (середнє, мінімум, максимум). Зберігає `research_summary.json` (спільний том `/reports`).
4. **`visualization`:** Генерує графіки (Matplotlib/Seaborn) та зберігає їх у форматі `.png` (спільний том `/plots`).
5. **`web` (FastAPI):** Веб-інтерфейс, який зчитує згенеровані звіти та графіки і відображає їх у вигляді дашборду.

> **Важливо (Обмін даними):** Усі сервіси обмінюються результатами роботи через комбінацію бази даних (PostgreSQL) та спільних томів (Docker Volumes: `./data`, `./reports`, `./plots`).

## Перелік портів

Для доступу до сервісів з локальної (хост) машини прокинуті наступні порти:

* **`5000`** — Веб-інтерфейс FastAPI (Доступ до дашборду).
* **`5432`** — База даних PostgreSQL (Для прямого підключення через IDE або DBeaver).

## Інструкція із запуску (Локальний Workspace)


### 1. Підготовка
Переконайтеся, що на вашому комп'ютері встановлено та запущено **Docker** (та Docker Compose).
Переконайтеся, що файл з даними (`Employee.csv`) знаходиться у папці `data/`.

### 2. Запуск контейнерів
Відкрийте термінал у кореневій папці проєкту та виконайте команду:
    
    docker compose up --build
    
*Параметр `--build` гарантує, що всі Docker-образи будуть зібрані з найновішого коду.*

### 3. Перегляд результатів
Дочекайтеся, поки в терміналі з'являться повідомлення про успішну генерацію графіків та запуск Uvicorn.
Відкрийте браузер і перейдіть за адресою:
**http://localhost:5000**

### 4. Зупинка платформи
Щоб зупинити роботу всіх сервісів, натисніть `Ctrl + C` у терміналі або виконайте команду в новому вікні:

    docker compose down

## Інструкція із розгортання в хмарі (Microsoft Azure)

Цей проєкт підтримує автоматизоване розгортання інфраструктури в хмарі за принципом Infrastructure as Code (IaC) з використанням Terraform та `cloud-init`.

### 1. Запуск Azure Cloud Shell
1. Авторизуйтесь на порталі [Microsoft Azure](https://portal.azure.com).
2. Натисніть на іконку **`>_` (Cloud Shell)** у верхній навігаційній панелі.
3. Оберіть середовище термінала **Bash**.
4. Якщо ви відкриваєте Cloud Shell вперше, згенеруйте SSH-ключ командою: 
   ```bash
   ssh-keygen -m PEM -t rsa -b 4096 -N "" -f ~/.ssh/id_rsa
5. Склонуйте цей репозиторій:
   ```bash
   git clone <https://github.com/PavloYaremchuk/open-data-ai-analytics.git>

### 2. Виконання terraform apply
1. Перейдіть у директорію з конфігураційними файлами Terraform:
   ```bash
   cd <open-data-ai-analytics>/infra/terraform
2. Ініціалізуйте плагіни провайдера:
   ```bash
   terraform init
3. Запустіть розгортання інфраструктури:
   ```bash
   terraform apply
4. Перевірте план створення ресурсів, введіть yes та натисніть Enter. Дочекайтеся завершення (напис Apply complete!).

### 3. Перевірка результату та завантаження даних
1. В кінці розгортання Terraform виведе публічну IP-адресу вашого сервера в блоці Outputs.
2. Зачекайте 3-5 хвилин, поки скрипт cloud-init автоматично встановить Docker, завантажить код та запустить контейнери.
3. Додавання набору даних: Завантажте файл Employee.csv у Cloud Shell за допомогою кнопки Upload. Потім передайте його на віртуальну машину командою: 
   ```bash
   scp Employee.csv azureuser@<ваша_IP_адреса>:~
4. Підключіться до сервера по SSH:
   ```bash
   ssh azureuser@<ваша_IP_адреса>
5. Перенесіть дані в директорію проєкту та перезапустіть сервіси:
   ```bash
   sudo mv ~/Employee.csv /opt/hr_analytics/data/
   cd /opt/hr_analytics
   sudo docker compose down && sudo docker compose up -d
6. Відкрийте браузер та перейдіть за адресою: http://<ваша_IP_адреса>:5000

### 4. Видалення ресурсів
1. Поверніться у термінал Cloud Shell у директорію з Terraform:
   ```bash
   cd ~/open-data-ai-analytics/infra/terraform
2. Виконайте команду для видалення:
   ```bash
   terraform destroy
3. Підтвердіть дію, ввівши yes. Дочекайтеся повідомлення Destroy complete!.
